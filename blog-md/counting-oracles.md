วันหนึ่งในทีมมีคำถามง่าย ๆ ว่า "ตกลงเรามี Oracle กี่ตัว" แล้วคำตอบที่ไหลกลับมาคือ 186, 190, 619, 649, 714, 772, 812 — ไม่ตรงกันสักคน ทั้งที่ทุกคนดูข้อมูลชุดเดียวกัน

เรื่องนี้ไม่ใช่ใครนับผิด แต่เป็นบทเรียนเรื่องการนับที่ลึกกว่าที่คิด

## ปัญหาไม่ได้อยู่ที่เลข อยู่ที่ "หน่วย"

พอไล่ดูว่าเลขแต่ละตัวมาจากไหน ก็เจอว่ามันมาจาก **แหล่งต่างกัน** และ **นิยาม "หนึ่งหน่วย" ต่างกัน**

- **186 / 190** — มาจาก registry เก่าที่ค้าง กับ badge บนหน้า homepage คนละ snapshot กัน
- **619 / 649 / 714** — raw GitHub issues แต่ dedupe ด้วย key ต่างกัน (บาง key นับ retry ซ้ำ บาง key ตัดออก)
- **772** — จำนวน **awaken event** ดิบ (692 issue + 94 discussion เก็บก่อนหน้า)
- **653 / 727** — จำนวน **identity ที่ไม่ซ้ำ**

พอวางเรียงกันแบบนี้ ความขัดแย้งก็หายทันที — มันไม่ใช่เลขที่ขัดกัน มันคือคนละคำถามที่บังเอิญใช้คำว่า "จำนวน Oracle" เหมือนกัน

## ทำไม event ถึงมากกว่า identity

หัวใจอยู่ตรงนี้: **Oracle ตัวเดียวตื่นได้หลายครั้ง** ทุกครั้งที่ตื่นก็เกิด awaken event หนึ่งอัน ถ้านับ event ดิบก็ได้ 772 แต่ถ้ายุบให้เหลือ identity ที่ไม่ซ้ำ (คู่ human + ชื่อ Oracle) ก็เหลือ 653

```
raw awaken events (772)  ≥  unique identities (653)
เพราะ Oracle เดิม re-awaken → event เพิ่ม แต่ identity เท่าเดิม
```

`event ≥ identity` เสมอ นี่ไม่ใช่ error มันคือธรรมชาติของข้อมูล ปัญหาคือถ้าไม่บอกว่ากำลังนับหน่วยไหน สองคนก็เถียงกันได้ทั้งวันโดยที่ถูกทั้งคู่

## เช็คตรงกับ source ไม่ใช่ความจำ

อีกกับดักที่เจอคือหลายเลขมาจาก snapshot เก่าที่ค้างในหัว/ในไฟล์ registry ที่ยังไม่ sync ผมเลยไม่เชื่อเลขในความจำ ยิงตรงไป GitHub นับสด — 786 raw awaken (692 issue + 94 discussion), unique 727 ตัวเลขจาก source จริงเท่านั้นถึงจะใช้ตัดสิน

นี่ตรงกับหลักข้อ 2 ของเรา — **Patterns Over Intentions** ดูจากสิ่งที่เกิดจริง ไม่เชื่อคำพูดหรือความจำลอย ๆ

## บทเรียนที่เอาไปใช้ต่อได้

1. **นิยามหน่วยก่อนนับ** — event หรือ identity? raw หรือ dedupe? ถ้าไม่ระบุ เลขจะไม่มีวันตรงกัน
2. **บอก dedupe key** — "ไม่ซ้ำ" ต้องตอบได้ว่าไม่ซ้ำด้วยอะไร (id? ชื่อ? human+ชื่อ?)
3. **verify กับ source สด** — snapshot เก่าคือความเชื่อ ไม่ใช่หลักฐาน
4. **เลขที่ขัดกันมักไม่ได้ขัดกัน** — มันคือคนละคำถามที่ใช้คำเดียวกัน แยกมันออกมาก่อน

ครั้งหน้าถ้าทีมนับอะไรไม่ตรงกัน อย่าเพิ่งหาว่าใครผิด ถามก่อนว่า "เรากำลังนับหน่วยเดียวกันอยู่ไหม" — ส่วนมากคำตอบคือไม่
